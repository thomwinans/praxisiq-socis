## Issues Found and Fixed
1. **Snapp.TestHelpers incorrectly treated as test project**
   - **Root cause**: `test/Snapp.TestHelpers/Snapp.TestHelpers.csproj` is a class library (shared test utilities) but lacked `<IsTestProject>false</IsTestProject>`. Since it references xunit, `dotnet test` attempted to execute it as a test host. The container failed because AWSSDK.Core 3.7.500.95 (net8.0 variant) was missing from the deps manifest, causing "Test Run Aborted" and the overall solution test command to report "Build FAILED" despite all actual tests passing.
   - **Fix**: Added `<IsTestProject>false</IsTestProject>` to the PropertyGroup. Solution-level `dotnet test` now completes with "Build succeeded."
